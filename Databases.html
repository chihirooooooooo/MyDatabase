Implementation

You are to store no more than a few records in memory at a time. If you are declaring an array of structs in memory and filling it with all the data - you are doing something wrong. The records are to be on disk and you are to read the files one record at a time. Remember, the records must be of fixed length! Sample code for seeking for a record in Java, C++, and Python are provided.

Menu of operations

1) Create New Database
Prompts the user for the name of a .csv file (e.g., "Fortune500". Reads and converts the matching .csv file into a pair of files: Fortune500.config, Fortune500.data. This can be specific to a csv file that has exactly the 6 fields in our datafile. You may assume that the records in the csv file are sorted in order of the primary key, the name of the company. Any previous version of those files are destroyed.
  • Fortune500.config: contains the number of records in the sorted data file. This will be the number of records in the input data, e.g., 500 It may also contain any other information you find helpful. You may hardcode the names and sizes of the fields in your program, but you could use the config file to describe the names, sizes of the fields in order, anything else you want.
  • Fortune500.data: contains the data records, one per line, with fixed size fields. The first field is the key for each record and the records are assumed to be in sorted order by the key. You may use any separator you want (or no separator).
    • 2) Open Database
    The user should be prompted for the prefix for an already created pair of files storing the database contents. (e.g., "Fortune500"). If another database is already open, the user should be instructed to close the current database first before opening a new one. The open method of the DB class should be called.
    3) Close Database
    The current database files should be closed.
    4) Display record
    You must find the record via the primary key using seeks and binary search, then display it. Each field should display the name (optionally from the config file) and the value (from the data file record).
    5) Update record
    As a first step, you have to find the record using the same "find" routine as 4. Then you should display the contents and allows updates in a specified field. Don't allow the updating of the primary key.
    6) Create report
    This should display the first ten records, to the screen, nicely formatted, in sorted order by primary key.
    7) Add a record: Required for pairs, 10% bonus for individuals
    Add the record at the database. This should only be attempted after everything else is working. Save that working version to go back to in case this messes up.
    1) Write the new record to the bottom of the file.
    2) Change CreateDB so that the config file stores 2 numbers: the number of sorted records (which will always be 500) and the number of overflow records which is initially 0.
    3) Read and write the config file to add 1 to the number of overflow records. 4) update find record to first do binarySearch on the sorted records (as before) but, if that fails, then do linear search starting on the overflow records (i.e., binarySearch records 0-499, linear search on records 500 to end of file).
    8) Delete a record
    Delete a record from the database. As records are deleted, they are logically removed from the file (most likely just overwrite them with a blank record).
